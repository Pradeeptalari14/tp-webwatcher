# 📖 Glossary — WebWatcher

> Key terms, acronyms, and concepts used in this studio and its generated configurations.

---

## General SRE & DevOps Terms

| Term | Definition |
|------|-----------|
| **SRE** | Site Reliability Engineering — applying software engineering principles to infrastructure and operations |
| **IaC** | Infrastructure as Code — managing infrastructure through machine-readable config files |
| **CI/CD** | Continuous Integration / Continuous Delivery — automated build, test, and deploy pipelines |
| **GitOps** | Using Git as the single source of truth for infrastructure and application state |
| **Idempotency** | Property where running an operation multiple times produces the same result |
| **Immutable Infrastructure** | Infrastructure that is replaced rather than modified when changes are needed |
| **Zero Trust** | Security model assuming no implicit trust — verify every request, every time |
| **Least Privilege** | Grant only the minimum permissions required to perform a task |
| **RBAC** | Role-Based Access Control — permissions based on roles, not individual users |
| **mTLS** | Mutual TLS — both client and server authenticate each other |
| **WAF** | Web Application Firewall — filters malicious HTTP traffic |
| **SAST** | Static Application Security Testing — analyzing source code for vulnerabilities |
| **DAST** | Dynamic Application Security Testing — testing running applications for vulnerabilities |
| **CVE** | Common Vulnerabilities and Exposures — publicly disclosed security vulnerabilities |
| **SLA** | Service Level Agreement — contracted uptime/performance commitment |
| **SLO** | Service Level Objective — internal target for reliability metrics |
| **SLI** | Service Level Indicator — measured metric (e.g., request success rate) |
| **Error Budget** | Allowable downtime derived from SLO (100% - SLO%) |
| **MTTR** | Mean Time To Recovery — average time to restore service after an incident |
| **MTTD** | Mean Time To Detect — average time to discover an issue |
| **Runbook** | Step-by-step operational guide for handling specific scenarios |
| **Postmortem** | Blameless analysis of an incident to prevent recurrence |
