## Reporting a Vulnerability

If you discover a **security vulnerability** in any configuration file, script, or documentation in this repository:

1. **Do NOT open a public GitHub issue.**
2. Email directly: **pradeeptalari14@gmail.com**
3. Include:
   - File path and line number
   - Description of the vulnerability
   - Potential impact
   - Steps to reproduce (if applicable)

**Expected Response Time:** Within 48 hours.

## Security Best Practices for This Studio

### ❌ Never Do This
```bash
# DON'T commit real credentials
AWS_SECRET_KEY=AKIAxxxxxxxxxxxxxxxx  # ← NEVER in git
DB_PASSWORD=mysecretpassword         # ← NEVER in git
API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxx    # ← NEVER in git
```

### ✅ Always Do This
```bash
# Use environment variables
export DB_PASSWORD=$(aws ssm get-parameter --name /prod/db-password --with-decryption --query Parameter.Value --output text)

# Use .env files (listed in .gitignore)
echo "DB_PASSWORD=secret" >> .env
echo ".env" >> .gitignore

# Use secret managers in production
# AWS SSM Parameter Store, HashiCorp Vault, GitHub Secrets, Azure Key Vault
```
